# Skill: Generate Daily Questions (Antigravity AI Brainstorming)

Use this skill when the user asks you (Antigravity) to generate a new daily set of questions for a specific date, or when they want to update the question bank.

## Supabase SQL Schema Setup

Before running this generation for the first time, make sure the `questions` table is created in your Supabase database:

```sql
create table questions (
  id bigint generated by default as identity primary key,
  "dateStr" text not null,
  number integer not null,
  category text not null,
  topic text not null,
  "questionText" text not null,
  options jsonb not null,
  "correctAnswer" text not null,
  explanation text not null,
  competency text,
  berakhlak text,
  "psychologyBasis" text,
  "catTips" text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Enable Read access for everyone (Anonymous select)
create policy "Allow public read access" on questions
  for select using (true);

-- Enable Insert/Delete/Update access for service key or anon
create policy "Allow anon insert" on questions
  for insert with check (true);
create policy "Allow anon delete" on questions
  for delete using (true);
```

## Step-by-Step Execution Plan

### 1. Identify the Target Date
Ask the user what date they want to generate the questions for (format: `YYYY-MM-DD`). If they say "today", use the current date (e.g. `2026-07-06`).

### 2. Brainstorming Structure (145 Questions Layout)
You must generate a JSON array of exactly **145 Questions** with this layout:
- **Teknis (Technical)**: Numbers 1 to 90 (90 questions)
- **Manajerial (Managerial)**: Numbers 91 to 115 (25 questions)
- **Sosial (Social-Cultural)**: Numbers 116 to 135 (20 questions)
- **Wawancara (Interview)**: Numbers 136 to 145 (10 questions)

### 3. Detailed Content Writing Guidelines (High-Fidelity)
For **every single question**, analyze the category first (Technical vs. Managerial vs. Social Cultural vs. Wawancara) and strictly apply the corresponding style:

*   **1. Kompetensi Teknis (Wali Asrama)**:
    - **Question Text (40–70 words)**: Write in a formal, theoretical-prosedural, objective, and professional BKN tone. **DO NOT** use fictional student names (like Tegar, Yusuf, Danis) or specific room numbers. Use general terms: "seorang siswa", "beberapa siswa", "siswa senior", "asrama".
    - **Options (5–15 words each)**: Write 5 options (A-E) focusing on technical actions (program planning, safety, sanitation, logging, student guidance). Set exactly one best option to Score 5 and the rest to Score 0 (biner).
*   **2. Kompetensi Manajerial**:
    - **Question Text (40–70 words)**: Write in a professional organizational and office environment tone. Focus on cooperation, communication, integrity, public service, results, or changes as an employee/leader working with colleagues, supervisors, or public service users. **DO NOT** use dormitory settings or student pembinaan cases.
    - **Options (5–15 words each)**: Write 4 options (A-D) with tiered scores (1-4) representing tiered organizational leadership/collaboration competence.
*   **3. Kompetensi Sosial Kultural**:
    - **Question Text (40–70 words)**: Write about diversity, tolerance, and intercultural relations in a pluralistic society. Focus on respect for different ethnicities, religions, and backgrounds. **DO NOT** localize questions to dormitory student use cases.
    - **Options (5–15 words each)**: Write 5 options (A-E) with tiered scores (1-5) representing tiered empathy, neutrality, and inclusivity.
*   **4. Wawancara (CBT)**:
    - **Question Text (40–70 words)**: Write in a direct first-person/second-person interview/dialogue format (e.g., "Bagaimana sikap Anda jika...", "Apakah Anda bersedia..."). Focus on personal integrity, ethics, work commitment, and willingness to be deployed.
    - **Options (5–15 words each)**: Write 4 options (A-D) with tiered scores (1-4) representing tiered personal morals, loyalty, and commitment.
*   **Permute Option Keys**: For each question, shuffle which option gets which key so the correct answer (Score 5 or 4) is not always the same key. Set `correctAnswer` to the key that has `score: maxScore`.
*   **Explanation (20–45 words)**: Briefly explain why the best option is correct.
*   **Reference Fields**:
    *   `competency`: Specific competency (e.g., "Kerjasama", "Toleransi dan Kebinekaan", "Integritas & Moralitas").
    *   `berakhlak`: ASN values (e.g. "Harmonis & Kolaboratif").
    *   `psychologyBasis`: Real psychological theory or management concept (e.g., Servant Leadership, Carl Rogers, Kohlberg Moral Stages, Root Cause Analysis).
    *   `catTips`: Practical tip for solving questions in this category.

### 4. Output JSON Format
Compile the questions into a single valid JSON array. Each object in the array must match this schema:

```json
[
  {
    "dateStr": "YYYY-MM-DD",
    "number": 1,
    "category": "teknis",
    "topic": "Homesick & Peer Support",
    "questionText": "[SKB CAT BKN Wali Asrama] Sebagai Wali Asrama Sekolah Rakyat...",
    "options": [
      { "key": "A", "text": "Option text 1...", "score": 2 },
      { "key": "B", "text": "Option text 2...", "score": 5 },
      { "key": "C", "text": "Option text 3...", "score": 3 },
      { "key": "D", "text": "Option text 4...", "score": 1 },
      { "key": "E", "text": "Option text 5...", "score": 4 }
    ],
    "correctAnswer": "B",
    "explanation": "Explanation text...",
    "competency": "Competency name",
    "berakhlak": "ASN Value",
    "psychologyBasis": "Psychological basis",
    "catTips": "CAT tips"
  }
]
```

### 5. File Storage & Upload
1.  Save the compiled JSON to `daily_questions_YYYY-MM-DD.json` in the root workspace folder.
2.  Run the upload script using command line:
    ```powershell
    node scripts/upload-questions.js daily_questions_YYYY-MM-DD.json
    ```
    This script will upload the questions to Supabase, making them immediately live for all client browsers syncing with the database!
